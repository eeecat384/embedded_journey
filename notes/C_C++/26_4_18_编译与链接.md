# 编译与链接

计算机语言的从图灵机时代二进制的机器码，到汇编语言，再到高级语言，其对人的可读性不断增强，对机器的可读性不断减弱，为了让计算机能读懂高级语言，需要通过编译器一步步翻译为机器语言

而链接器的职能便是将不同源文件之间的代码联系起来

## 0. 编译过程概述

C/C++程序从源代码到可执行文件，有四个阶段：

.c → [预处理] → .i → [编译] → .s → [汇编] → .o → [链接] → .exe

## 1. 预处理（Preprocessing）

预处理阶段主要进行对伪指令（#开头指令）和特殊符号处理：

1. 宏定义：如`#difine N 10`就是将源代码中所有的`N`替换为`10`，`#undef N`就是解除这行之后源代码对N的替换
2. 条件编译指令：如`#ifndef` `#else` `#elif` `#endif`等，这些伪指令能让开发人员可以通过定义不同的宏来决定编译器对哪些代码进行处理
3. 头文件包括语句：如`#include <iostream>` `#include "log.h"`，编译器会直接将include的文件内容直接复制到对应位置<br>使用<>指定的是cpp或c的标准库，编译器会直接在标准库路径搜索相关文件，如果使用""指定文件，编译器会按照`当前文件目录->项目指定包含目录->标准库路径`的顺序搜索
4. 特殊符号：如`__LINE__` `__LINE__`
    ```c
    #include <stdio.h>
    int main(){
    printf("当前文件: %s\n", __FILE__);   // 当前文件: D:\...\demo.c
    printf("当前行号: %d\n", __LINE__);   // 当前行号: 4
    printf("编译日期: %s\n", __DATE__);   // 编译日期: Apr 18 2026
    printf("编译时间: %s\n", __TIME__);   // 编译时间: 14:14:57
    printf("当前函数: %s\n", __func__);   // 当前函数: main
    }
    ```

## 2. 编译（Compilation）

编译阶段，编译器会将代码用下面的步骤翻译为汇编语言：

1. 词法分析：将代码拆分为token（关键字、标识符、运算符等）
2. 语法分析：构建语法树，检查语法错误
3. 语义分析：类型检查、作用域分析
4. 代码优化：优化中间代码，去掉不必要的语句等
5. 生成汇编：输出平台相关的汇编指令

不同的编译器其编译的细节会有所不同，但大体遵循这套流程

## 3. 汇编（Assembly）

汇编阶段的主要任务主要是将生成的汇编代码翻译为机器码，翻译后的文件为二进制文件

## 4. 链接（Linking）

链接阶段，链接器会将不同源文件的变量，函数等联系到一起

比如有main.cpp和log.cpp

```cpp
//log.cpp
#include <iostream>

void log(const char* msg){
    std::cout << msg << endl;
}
```

```cpp
//main.cpp
#include <iostream>

void log(const char* msg);

int add(int a, int b){
    log("test")
    return a + b;
}

int main(){
    add(5, 8)
}
```

在编译器将两个文件编译后，链接器会根据main.cpp的函数声明找到log.cpp中的函数定义

如果log中没有对应函数定义，如我们将log.cpp改成：

```cpp
//log.cpp
#include <iostream>

void notlog(const char* msg){
    std::cout << msg << endl;
}
```

会显示报错：`cppdemo.obj : error LNK2019: 无法解析的外部符号 "void __cdecl log(char const *)" (?log@@YAXPEBD@Z)，函数 "int __cdecl add(int,int)" (?add@@YAHHH@Z) 中引用了该符号`

报错`LNK2019`说明这个错误是在链接过程中出错的（对应的，C开头错误码是编译过程出现错误）

如果我们将add函数中的log函数注释掉，即log函数在main.cpp中没有被执行，则程序正常运行，没有错误

一个相似的思路是，我们只注释掉main函数中的add函数调用，即：

```cpp
//log.cpp
#include <iostream>

void notlog(const char* msg){
    std::cout << msg << endl;
}
```

```cpp
//main.cpp
#include <iostream>

void log(const char* msg);

int add(int a, int b){
    log("test")
    return a + b;
}

int main(){
    //add(5, 8)
}
```

通常来想，没有调用add函数，自然也没有调用log函数，则程序应该正常运行

可实际情况还是有和上面相同的LNK2019错误，这是因为在这个文件里，我们不使用add函数，但技术上我们可能在另一个文件中使用它，所以链接器确实需要链接它

如果在add函数前加上static，相当于告诉链接器：我只会在这个文件中使用add函数。这样，程序就能正常运行了

另一种常见的链接错误是：有重复的符号，比如log.cpp内容为：

```cpp
//log.cpp
#include <iostream>

int log(const char* msg){     //返回值类型变为int
    std::cout << msg << endl;
    return 0;
}
```

```cpp
//log.cpp
#include <iostream>

void log(const char* msg， int a){    //参数多了int a
    std::cout << msg << endl;
}
```

因为此时main.cpp中的函数声明还是`void log(char* msg)`，所以这两种声明都会导致链接报错

同样的，如果有两个返回值，函数名，传入参数都相同的函数，链接器也会报错，比如在下面的例子中

```h
//log.h
void notlog(const char* msg){
    std::cout << msg << endl;
}
```

```cpp
//math.cpp
#include <iostream>
#include "log.h"

int add(int a, int b){
    log("test0");
    return a + b;
}
```

```cpp
//main.cpp
#include <iostream>
#include "log.h"

int add(int a, int b);

int main(){
    add(5, 8);
}
```

因为`#include`实际上是将文件内容复制粘贴，所以在两个cpp源文件中出现了两个log函数的定义，这会引起链接器的报错

有下面几种解决方式：
1. 在log函数定义加上前缀static，这使得在多个翻译单元中的log定义不会互相链接
2. 在log函数定义加上前缀inline，这会将函数调用语句替换成函数体
3. 将log定义放在翻译单元(cpp源文件)而不是.h文件中，.h中只放函数声明
